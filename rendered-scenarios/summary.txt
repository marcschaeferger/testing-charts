## Known Issues

### Chart Template Bug (09-mtls-pkcs12)
The original chart template at `charts/newt/templates/deployment.yaml` has an indentation bug that causes YAML parsing errors when rendering with certain configuration combinations:
- Line 48-49: Inconsistent indentation in podLabels range loop
- Line 52: Inconsistent indentation in podAnnotations range loop  
- Line 56-62: Issues with annotations handling causing "block sequence entries not allowed" error

This affects scenarios where:
- `global.metrics.annotations` is nil/undefined (default)
- Instance has mtls.enabled=true with mode=pkcs12

The bug appears to be triggered by the Helm templating engine processing the annotations section incorrectly under specific value combinations. The issue is in the original chart and needs to be fixed by the chart maintainers.

## Rendering Commands

```bash
# Render all scenarios
for f in rendered-scenarios/values/*.yaml; do
  name=$(basename "$f" .yaml)
  helm template newt ./charts/newt -f "$f" --namespace newt-test > "rendered-scenarios/manifests/$name.yaml"
done

# Validate all
for f in rendered-scenarios/manifests/*.yaml; do
  kubeconform -strict -summary "$f"
done
```
